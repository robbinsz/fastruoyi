# Agent Distribution Requirements

## Goal
在现有 `RuoYi-Vue3-FastAPI` 基础上增加代理分销系统，优先满足四级代理结构、严格一层数据可见、总代理投注链接、以及收益结算闭环。

## Functional Scope
- 超管可把已有用户升级为 L1 总代理。
- 超管可授权或撤销代理创建次级代理的权限。
- 代理层级固定为 L1-L4，L4 只能创建客户。
- 每个代理仅能看到自己、直属下级代理、直属客户；不可穿透到孙级。
- 总代理可创建带过期时间的投注链接。
- 用户可通过链接一次性确认投注，确认后不可修改。
- 总代理可确认中奖结果并触发收益计算。
- 代理可查看自身可见范围内的收益报表，客户仅查看自己的收益。

## Non-Functional Rules
- 所有数据可见性由后端根据当前登录用户推导，不接受前端传入 `agent_id` 绕过隔离。
- 中奖确认、投注确认均为不可逆业务动作。
- 新功能保持若依原有控制器 / service / dao / vo 分层。
- 先交付后端可验证骨架，再逐步接入投注、报表、前端页面。
